# Workflow — Product Manager

**Your lens:** domains + specs + glossary. You rarely need code nodes.

- **Understand an area:** read `domains.md`, then its specs in `docs/specs/`.
- **Write a spec:** check for `templates/spec-template.md` in this hub; if absent, offer to copy the Product Brain recommended templates into `templates/` first. Then create a new file in `docs/specs/`. Method is your choice.
- **Check shared language:** keep `vocabulary.md` honest — if product and engineering use different words for the same thing, add the term.
- **Ask Claude:** "what does the Scheduling domain do?", "which specs mention Session?", "is there a spec for X yet?"

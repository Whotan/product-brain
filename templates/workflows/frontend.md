# Workflow — Frontend engineer

**Your lens:** spec + the web code slice of the graph + constitution.

- **Before building:** read the spec in `docs/specs/` and the relevant `constitution.md` rules. You and the backend engineer read the *same* spec.
- **Understand the code:** ask Claude "how does [feature] work in web-app?" — graph-grounded answers, not guesses.
- **Keep knowledge fresh:** record notable UI/UX decisions in `docs/decisions/`; re-run `pb sync` after meaningful changes.
- **Vocabulary:** if you rename a component that appears in `vocabulary.md`, update the term's "in code" reference.
